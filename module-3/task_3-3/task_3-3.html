1. What is the difference between invoking and creating a function?

    2. Creating a function means writing its definition (what the function does). Invoking a function means calling it so the code inside runs.

  2. Explain the syntax regarding semicolons and code blocks.

    3. Semicolons end single statements in JavaScript.Code blocks use {} and group multiple statements together, for example in functions or loops.

  3. What happens when we try to store values from a function when it doesn't return any?

    4. If a function doesn't return a value, storing its result will result in the value `undefined`.

  4. How do we separate parameters from each other when creating a function?

    5. Parameters are separated by commas inside the parentheses in the function definition.

  5. Explain how parameter names are related during the invocation and creation of a function.

    6. The names used when creating a function are local to that function. When invoking the function, values are passed by position.

  6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. If a function call is missing a parameter, the corresponding parameter in the function definition will be undefined because no value was provided for it.

  7. Explain what happens when we use recursive functions.

    8. A recursive function calls itself to solve a smaller part of a problem. It must have a stopping condition, or it will run forever and cause an error.
